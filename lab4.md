# Лабораторная работа №4: ETL-конвейеры на Dask

**Вариант 7**  
**Используемый датасет:** `Parking_Violations_Issued_-_Fiscal_Year_2014__August_2013___June_2014_.csv`

## Цель работы

Получить практические навыки построения ETL-пайплайнов (Extract, Transform, Load) с помощью библиотеки Dask для обработки больших объёмов данных, не умещающихся в оперативной памяти. Разобраться с концепцией «ленивых вычислений», управлением памятью и визуализацией ориентированных ациклических графов (DAG).

## Выводы по работе

В ходе выполнения работы я приобрёл практические навыки работы с Dask для создания ETL-конвейеров при обработке больших массивов данных.

- **Ленивые вычисления:** я на практике убедился, что операции чтения, удаления столбцов, приведения типов и создания новых признаков в Dask не выполняются мгновенно – они лишь формируют граф вычислений (DAG), что позволяет эффективно управлять ресурсами.
- **Разработанный ETL-пайплайн:** я полностью реализовал конвейер, включающий этапы извлечения данных из CSV, их трансформации (очистка, типизация, создание новых признаков) и загрузки в результирующий формат.

Работа помогла мне понять, как Dask справляется с датасетами, превышающими доступную память, и почему ленивые вычисления критичны для масштабируемой обработки данных.
